# E2E Test Plan

## Overview
Two-layer test strategy:
1. **Backend API integration tests** (xUnit + WebApplicationFactory) — fast, no Docker, mocks the agent
2. **Full-stack E2E tests** (Playwright) — runs against Docker Compose, tests real user flows end-to-end

## Verification Strategy
- Backend: `dotnet test` from `backend/`
- Full-stack: `npx playwright test` from `tests/e2e/`
- Both run independently

---

## Task 1: Backend Test Project Setup

### Create `backend/tests/KnowledgeEngine.Api.Tests/`
- xUnit test project targeting .NET 8
- Reference `KnowledgeEngine.Api.csproj`
- Install `Microsoft.AspNetCore.Mvc.Testing` for `WebApplicationFactory`
- Install `Microsoft.EntityFrameworkCore.InMemory` for test DB
- Add to solution file

### Test infrastructure
- `CustomWebApplicationFactory` : `WebApplicationFactory<Program>`
  - Replaces SQLite with `InMemoryDatabase` (unique name per test)
  - Replaces `IAgentService` with `MockAgentService`
  - Sets `Library:Path` to a temp directory
  - Replaces Hangfire with `BackgroundJobClient` mock (no actual job processing)
- `MockAgentService` : `IAgentService`
  - Records calls for assertion
  - `EnsureSessionAsync` returns a fake session ID
  - `SendPromptAsync` records the prompt
  - `GetSessionInfoAsync` returns configurable `SessionInfo`
  - `KillSessionAsync` no-op

### Acceptance Criteria
- `dotnet test` runs and passes a single smoke test
- WebApplicationFactory creates the app without errors
